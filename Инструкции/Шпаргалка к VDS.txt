---

## 📋 Шпаргалка по управлению TrAIner Bot (VDS)

Все команды вводятся после подключения к серверу: `ssh root@188.120.246.129`.

### 1. Основное управление (Служба в фоне)

Бот настроен как системная служба, чтобы он работал 24/7 даже после закрытия консоли.

* 
**Перезапустить:** `systemctl restart trainer_bot` (делай это **всегда** после загрузки нового кода через FileZilla).


* 
**Проверить статус:** `systemctl status trainer_bot` (проверить, нет ли ошибок и горит ли «active»).


* 
**Остановить/Запустить:** `systemctl stop trainer_bot` / `systemctl start trainer_bot`.



---

### 2. Работа с виртуальным окружением (Venv)

В Ubuntu 24.04 библиотеки ставятся только внутри «контейнера». Используй это для установки новых пакетов (например, `faster-whisper`).

* **Войти в окружение:**
```bash
cd /root/TrAIner-bot && source venv/bin/activate

```


* **Установить новые библиотеки:** (только после входа в venv)
```bash
pip install -r requirements.txt

```


* 
**Выйти из окружения:** `deactivate`.



---

### 3. Просмотр логов (Дебаг)

Если бот не отвечает, посмотри, что происходит у него «в мозгах» в реальном времени.

* 
**Следить за логами:** `journalctl -u trainer_bot -f`.


* 
**Выход из логов:** Нажми `Ctrl + C`.



---

### 4. Обновление кода (через FileZilla или Git)

1. Загрузи файлы через **FileZilla** (или используй `git pull` в папке `/root/TrAIner-bot`).


2. Если добавил новые библиотеки, обнови их через **Venv** (см. пункт 2).


3. 
**Обязательно** примени изменения: `systemctl restart trainer_bot`.



---

### 5. Ручной запуск (для поиска ошибок)

Если через `systemctl` бот падает, запусти его «вручную», чтобы увидеть текст ошибки прямо в консоли:

1. 
`systemctl stop trainer_bot`.


2. 
`cd /root/TrAIner-bot`.


3. 
`source venv/bin/activate`.


4. 
`python3 main.py`.


5. *(Для выхода нажми `Ctrl + C` и не забудь потом снова включить службу через `systemctl start`)*.

---

### 📂 Важные пути:

* 
`/root/TrAIner-bot/` — корень проекта.


* 
`/root/TrAIner-bot/bot_database.db` — файл базы данных (самое ценное, делай бэкап!).


* 
`/root/TrAIner-bot/.env` — здесь лежат твои API ключи и токены.
